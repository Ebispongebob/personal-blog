用 Claude Code 攒项目的时候，agent 经常要跑十几分钟。盯着进度条看两秒，手就会不受控地切到微信。

公众号、博客、再一个公众号。等回过神，进度到 90% 了，这十分钟刷了七八篇。

但一回忆，脑子里一片空白。标题记得几个，内容一个字都复述不出来。

这事最近反复发生。每次 agent 一跑长任务，我就条件反射往信息流里扎。刷的时候挺舒服，停下来什么都没留下。

后来琢磨了一下，发现这其实不是信息获取的问题，是等待焦虑。

## 等待为什么难受

把任务交给 agent，等于把控制权交出去了。任务出去了，结果没回来，人就悬在半空。手上没活，心里没底，进度条又戳不动。

这种悬置的状态特别难熬。

人最难熬的往往不是忙，是等。忙的时候被事情推着走，没空焦虑；等的时候只能跟自己脑子里的念头待在一起，那才折磨。所以身体替你找出口，刷点东西填一下。

碎片阅读最妙的地方是即时满足。每滑过一篇，大脑挤一点多巴胺，三秒钟，刚好够你不那么难受。滑完一篇焦虑还在，因为 agent 还在跑，再滑一篇。循环。

这跟信息本身没关系，就是馋那点多巴胺来压住等待的难受。想明白这点之后我有点后怕，一直以为自己在「利用碎片时间学习」，其实是在嗑信息止痛药。

## 碎片为什么读不进去

一篇能让你记住的内容，得挂在一个钩子上。钩子是你脑子里已有的问题、已有的场景、已有的那点模糊的痒。新信息挂上去，知识树长出一片叶子，它才是你的。

碎片阅读不给你钩子。那七八篇文章，从 AI 落地到商业八卦到某个大佬发言，互相之间毫无关系。大脑接收到的是一堆孤立散落的信号，没有东西能把它们串起来。

而且刷的时候没有输出。不写，不复述，不跟人聊。信息进来又流出去，像水过门石，什么也没沉淀。

所以「读了跟没读一样」，不是专注力不行，是这种阅读形态本身的缺陷。它是个流水槽，不是仓库。

## 三个还算管用的办法

我自己踩这个坑踩了挺久，摸索出几个还算管用的办法。说不上完全跑通，有时候还是会破功，但比以前好多了。

**把等待变成检查点。** agent 一开跑，别盯着它，也别刷信息流，同时开一件有闭环的小事——写一段要交的代码、整理一份笔记、读一篇一直没读完的长文。agent 出结果就切回去，没出就继续。关键是这件事得有头有尾，能让你看到「我做完了这么一坨」。上次 Claude Code 跑重构的十几分钟，我把一份拖了一周的笔记整理完了，那种一石二鸟的踏实感，比刷七八篇公众号强太多。

**刷完逼自己一句话总结。** 看完一篇，停下来，用一句话讲它解决了什么问题、对你有什么用。说不出来就是没看进去。我数过，随手刷的公众号里能一句话总结出来的不到三成，剩下七成都是噪音。逼自己总结这一下，本身就是把信息挂上钩的动作。

**碎片只做种子，别做饲料。** 看到有意思的标题别当场读，丢进收藏，等有完整的一块时间集中读，读的时候写两句自己的判断。碎片信息加碎片时间等于饲料，碎片信息加完整时间加输出，才叫知识。

## 碎片在反过来切碎你

本来等 agent 的那十分钟是完整的一块。你拿碎片信息去填，信息是碎的，注意力就跟着被切成碎的。等 agent 出结果，结果回来了，可大脑已经不在能进入深度的状态里，被那七八篇文章切得七零八落，得花好一会儿才能重新进入。

你以为赢了那十分钟，其实输掉的是后面那一段连续的专注。

这是碎片时代最阴的一点。它不偷你的时间，偷的是时间的形状。把完整的切成碎的，把碎的填满，让你时刻有事干、时刻不寂寞，但也时刻进不了深。

再往深一层。这个时代所有人都在教你怎么填满时间、怎么每一秒都不浪费、怎么并行三件事。很少有人讲，有些时间就该空着。

神经科学里有个东西叫默认模式网络，是大脑不在专注做某件事、处于放空发呆时最活跃的那套网络。创造性联想、把不相关的点连起来、突然想明白一件事的时刻，大多发生在它活跃的时候。也就是说，大脑最值钱的一个工作模式，恰恰需要你闲着、空着才能启动。

拿碎片信息把所有空档填满，等于把这个网络关了。你以为在高效利用时间，其实是在亲手关掉大脑里最贵的那台机器。

## 让等待就是等待

等待被严重低估了。我们把它当成要消灭的难受，用多巴胺止痛。可它其实是大脑从前一个任务切出来、准备进入下一个任务的过渡带。用信息把它压下去，过渡带就被跳过了。下一个任务进得去才怪。

所以现在再开 agent 跑长任务，我会做一件事：把微信窗口最小化，把公众号标签关掉，那十分钟就让它空着。要么干一件有闭环的小事，要么真空着，发个呆，倒杯水，看看窗外。

空着的那段时间，往往是一天里最值钱的几分钟。因为没东西填进来，脑子里那些一直被压着的念头才会冒出来。

agent 帮我把活干了。我得留点时间，让我自己也活着。
